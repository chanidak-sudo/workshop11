Workshop 11 : Material 3 Theme & ColorScheme Lab

เรียนรู้วิธีออกแบบสีของแอปทั้งระบบให้สอดคล้องกันด้วย ThemeData และ ColorScheme.fromSeed พร้อมรองรับโหมดสว่าง/มืดอัตโนมัติตามหลัก Material 3

Learning Content

หลัก Material Design กับการคุมโทนสีทั้งแอปด้วย ColorScheme

ในเวิร์กช็อปนี้นักศึกษาจะได้เข้าใจว่าทำไม Flutter เวอร์ชันใหม่ถึงไม่ให้กำหนดสีปุ่ม สีพื้นหลัง หรือสีตัวอักษรแบบกระจัดกระจาย แต่ใช้แนวคิด Single Source of Truth ผ่าน ThemeData + ColorScheme เพียงจุดเดียว และระบบจะนำสีไปใช้กับวิดเจ็ตทุกตัวอัตโนมัติ ทั้งโหมดสว่างและโหมดมืด

คอนเซปต์หน้าที่ในแอปจริง
ThemeDataกล่องเก็บค่าสไตล์ทั้งหมดของแอป — สี ตัวอักษร รูปร่างปุ่ม ความโค้งของขอบ
ColorScheme.fromSeedผลิตชุดสีครบ 30+ ตัว (primary, surface, onPrimary, error, ฯลฯ) จากสีหลักเพียงสีเดียว
brightnessบอกระบบว่าเป็นโทนสว่าง (Brightness.light) หรือมืด (Brightness.dark)
themeModeสวิตช์ระดับ MaterialApp ใช้ค่า ThemeMode.light / dark / system
Theme.of(context)คำสั่งดึงธีมปัจจุบันมาใช้ในวิดเจ็ตลูกได้ทุกระดับชั้น

Step 2

ตั้งค่า ThemeData ระดับแอปด้วย ColorScheme.fromSeed

ล้างโค้ดเดิมใน lib/main.dart ออกทั้งหมด แล้วเขียนสถาปัตยกรรมระบบใหม่ตามด้านล่าง

ข้อควรระวัง: ใน Material 3 ห้ามฮาร์ดโค้ดสีเช่น Colors.blue ใส่เข้าไปตรง ๆ ที่วิดเจ็ตทุกตัว เพราะจะไม่ตอบสนองต่อโหมดมืด ให้อ่านค่าผ่าน Theme.of(context).colorScheme เสมอ
lib/main.dart

import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatefulWidget {
  const MyApp({super.key});

  @override
  State createState() => _MyAppState();
}

class _MyAppState extends State {
  ThemeMode mode = ThemeMode.light;

  // ✅ จุดแก้ 1: ย้าย seedColor มาเป็น "ตัวแปร state" (ไม่ใช่ const อีกต่อไป)
  // เพื่อให้เปลี่ยนค่าได้ตอนรันไทม์แล้วทั้งแอปวาดใหม่
  Color seedColor = const Color(0xFF0F766E);

  void toggleMode() {
    setState(() {
      mode = mode == ThemeMode.light ? ThemeMode.dark : ThemeMode.light;
    });
  }

  // ✅ จุดแก้ 2: ฟังก์ชันเปลี่ยน seed ใหม่ + ฟังก์ชันรีเซ็ต
  // setState คือคำสั่งบอก Flutter ว่า "ค่าข้อมูลเปลี่ยนแล้ว ช่วยรีบิลด์ด้วย"
  void changeSeed(Color newSeed) {
    setState(() => seedColor = newSeed);
  }

  void resetSeed() {
    setState(() => seedColor = const Color(0xFF0F766E));
  }

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      themeMode: mode,
      theme: ThemeData(
        useMaterial3: true,
        colorScheme: ColorScheme.fromSeed(
          seedColor: seedColor, // ✅ ใช้ตัวแปร state แทนค่าคงที่
          brightness: Brightness.light,
        ),
      ),
      darkTheme: ThemeData(
        useMaterial3: true,
        colorScheme: ColorScheme.fromSeed(
          seedColor: seedColor, // ✅ ใช้ตัวแปร state แทนค่าคงที่
          brightness: Brightness.dark,
        ),
      ),
      home: HomePage(
        onToggleMode: toggleMode,
        isDark: mode == ThemeMode.dark,
        onSeedSelected: changeSeed, // ✅ จุดแก้ 3: ส่ง callback ลงไปให้หน้าจอลูก
        onResetSeed: resetSeed,
      ),
    );
  }
}
อธิบายโค้ดทีละบรรทัด (กดเพื่อศึกษาเอง)
StatefulWidgetใช้ StatefulWidget เพราะแอปต้อง "จำ" ได้ว่าตอนนี้อยู่โหมดสว่างหรือมืด — ค่าที่เปลี่ยนได้ระหว่างรันต้องเก็บใน State
modeตัวแปร ThemeMode mode คือสวิตช์กลางที่บอกทั้งแอปว่าจะแสดงโทนไหน เริ่มต้นที่ ThemeMode.light
toggleModeเมธอดสลับค่าโหมด สังเกตว่าต้องห่อด้วย setState(() {...}) เสมอ — มันคือคำสั่งบอก Flutter ว่า "ข้อมูลเปลี่ยนแล้ว ช่วยวาดหน้าจอใหม่ด้วย"
themeModethemeMode: mode ผูกตัวแปรของเราเข้ากับ MaterialApp โดยตรง เมื่อ mode เปลี่ยน Flutter จะสลับระหว่าง theme กับ darkTheme ให้อัตโนมัติ
theme / darkThemeเรากำหนด 2 ธีม โครงเหมือนกันต่างแค่ brightness — ตัวหนึ่ง Brightness.light อีกตัว Brightness.dark
fromSeedColorScheme.fromSeed(seedColor: ...) คือพระเอกของบทนี้ ใส่สีหลักไปสีเดียว ระบบจะคำนวณสีลูกอีก 30+ ตัวให้ครบ (primary, surface, error, onPrimary ...) โดยรับประกันว่าอ่านออกทั้งสองโหมด
homeส่งทั้งฟังก์ชัน toggleMode และสถานะ isDark ลงไปให้หน้าจอลูก เพื่อให้ปุ่มในแอปบาร์เรียกใช้ได้ — นี่คือหลักการ "ส่ง state ลงไปให้ลูก"

class HomePage extends StatelessWidget {
  const HomePage({
    super.key,
    required this.onToggleMode,
    required this.isDark,
    required this.onSeedSelected, // ✅ รับ callback เปลี่ยน seed
    required this.onResetSeed,
  });

  final VoidCallback onToggleMode;
  final bool isDark;
  final ValueChanged onSeedSelected; // ฟังก์ชันที่รับ "สี" 1 ตัว
  final VoidCallback onResetSeed;

  @override
  Widget build(BuildContext context) {
    final scheme = Theme.of(context).colorScheme;

    return Scaffold(
      backgroundColor: scheme.surface,
      appBar: AppBar(
        title: const Text('Theme & ColorScheme Lab'),
        backgroundColor: scheme.primary,
        foregroundColor: scheme.onPrimary,
        actions: [
          IconButton(
            icon: Icon(isDark ? Icons.light_mode : Icons.dark_mode),
            onPressed: onToggleMode,
          ),
        ],
      ),
      // ใช้ ListView แทน Column กันเนื้อหาล้นจอเวลาเพิ่มของ
      body: ListView(

        children: [
          Card(
            color: scheme.primaryContainer,
            child: Padding(
              padding: const EdgeInsets.all(20),

                crossAxisAlignment: CrossAxisAlignment.start,
                children: [
                  Text(
                    'สวัสดี Material 3',
                    style: TextStyle(
                      fontSize: 22,
                      fontWeight: FontWeight.bold,
                      color: scheme.onPrimaryContainer,
                    ),
                  ),
                  const SizedBox(height: 8),
                  Text(
                    'สีทุกตัวบนหน้านี้มาจาก ColorScheme เดียวกัน',
                    style: TextStyle(color: scheme.onPrimaryContainer),

                ],
              ),
            ),
          ),
          const SizedBox(height: 16),
          Row(
            children: [
              Expanded(
                child: ElevatedButton.icon(
                  onPressed: () {},

              ),
            ],

            onPressed: () {},
            child: const Text('Outlined Button'),

          Row(
            mainAxisAlignment: MainAxisAlignment.spaceBetween,
            children: [
              Expanded(
                child: Text(
                  'ตัวอย่างสีจาก ColorScheme (แตะเพื่อใช้เป็น seed)',
                  style: TextStyle(
                    fontWeight: FontWeight.bold,
                    color: scheme.onSurface,
                  ),
                ),
              ),
              TextButton(onPressed: onResetSeed, child: const Text('รีเซ็ต')),
            ],
          ),
          const SizedBox(height: 12),
          // ✅ จุดแก้ 4: แต่ละแถบสีส่ง onTap ที่ "เอาสีของตัวเองไปเป็น seed ใหม่"
          _SwatchRow(
            label: 'primary',
            color: scheme.primary,
            onColor: scheme.onPrimary,
            onTap: () => onSeedSelected(scheme.primary),

class _SwatchRow extends StatelessWidget {
  const _SwatchRow({
    required this.label,
    required this.color,
    required this.onColor,
    this.onTap, // ✅ เพิ่ม callback (ใส่ ? ได้ จะแตะหรือไม่แตะก็ได้)
  });

  final String label;
  final Color color;
  final Color onColor;
  final VoidCallback? onTap;

  @override
  Widget build(BuildContext context) {
    return Padding(
      padding: const EdgeInsets.only(bottom: 6),
      // ✅ ต้องมี Material + InkWell ถึงจะ "แตะได้" และมี ripple effect
      child: Material(
        color: color,
        borderRadius: BorderRadius.circular(8),
        child: InkWell(
          onTap: onTap,
          borderRadius: BorderRadius.circular(8),
          child: Padding(
            padding: const EdgeInsets.symmetric(horizontal: 14, vertical: 12),
            child: Row(
              mainAxisAlignment: MainAxisAlignment.spaceBetween,
              children: [
                Text(
                  label,
                  style: TextStyle(color: onColor, fontWeight: FontWeight.w600),
                ),
                Text(
                  'แตะเพื่อใช้เป็น seed →',
                  style: TextStyle(color: onColor, fontSize: 11),
                ),
              ],
            ),
          ),
        ),
      ),
    );
  }
}
อธิบายโค้ดทีละบรรทัด (กดเพื่อศึกษาเอง)
StatelessWidgetHomePage เป็น StatelessWidget ได้ เพราะมันไม่ได้ "เป็นเจ้าของ" สถานะใด — มันแค่รับค่ามาแสดงและส่งต่อเหตุการณ์กลับขึ้นไป
onToggleMode / isDarkfield สองตัวนี้รับมาจากแม่ (_MyAppState) ตัวแรกคือฟังก์ชันที่จะถูกเรียกเมื่อกดปุ่ม อีกตัวบอกว่าตอนนี้มืดอยู่ไหม (ไว้เลือกไอคอน)
schemefinal scheme = Theme.of(context).colorScheme; คือบรรทัดที่สำคัญที่สุด — มันดึงชุดสีของธีมปัจจุบันมาเก็บไว้ จากนี้ทุกสีบนหน้าจอจะอ้างจาก scheme ไม่มีการพิมพ์สีดิบเลย
backgroundColorพื้นหลัง Scaffold ใช้ scheme.surface / AppBar ใช้ scheme.primary และตัวอักษรบนแอปบาร์ใช้ scheme.onPrimary — สังเกตคู่ primary ↔ onPrimary ที่ออกแบบให้ตัดกันพอดี
IconButtonปุ่มในแอปบาร์ onPressed: onToggleMode เมื่อกดจะไปเรียกฟังก์ชันที่แม่ส่งมา ทำให้แม่สลับโหมด แล้วทั้งแอปวาดใหม่
Card / ButtonCard ใช้ scheme.primaryContainer เป็นพื้น และ onPrimaryContainer เป็นตัวอักษร ส่วนปุ่ม ElevatedButton/FilledButton ไม่ต้องกำหนดสีเอง — มันหยิบจากธีมให้อัตโนมัติ
_SwatchRowวิดเจ็ตย่อยที่เราแยกออกมาเพื่อแสดงตัวอย่างสีแต่ละช่อง รับ color + onColor เข้ามาเป็นคู่ พิสูจน์ว่าตัวอักษรอ่านออกบนพื้นสีนั้นจริง
FABFloatingActionButton ปุ่มกลมมุมขวาล่าง ก็ได้สีจากธีมเช่นกัน ในเวิร์กช็อปนี้ยังไม่ผูกการทำงาน (onPressed: () {}) ไว้ให้นักศึกษาต่อยอด

จุดที่กดได้ (Hotspots)

1
ปุ่มสลับโหมด (มุมขวาบน 🌙/☀️) — กดแล้วทั้งหน้าจอเปลี่ยนระหว่างโหมดสว่าง/มืด นี่คือผลของ onToggleMode ที่เรียก setState
2
ปุ่ม Elevated / Filled / Outlined — ปุ่มทั้ง 3 แบบของ Material 3 สังเกตว่าสีของมันมาจากธีมโดยไม่ได้กำหนดเอง
3
การ์ดสีฟ้าเขียว ใช้คู่สี primaryContainer + onPrimaryContainer ที่อ่านออกทั้งสองโหมด
4
แถบสี 4 ช่อง (primary / secondary / tertiary / error) — กดได้! กดช่องไหนก็จะเอาสีนั้นไปเป็น seedColor ใหม่แล้วสร้าง ColorScheme ทั้งชุดใหม่ทันที เหมือนที่ Step 4 ให้ลองเปลี่ยน seed (มีปุ่ม ↻ รีเซ็ตกลับค่าเดิม)
5
ปุ่ม + มุมขวาล่าง (FAB) — ในโค้ดยังไม่ผูกงาน เป็นจุดที่นักศึกษาต่อยอดได้

Lab Questions & Assignment

แบบฝึกหัดทบทวนและประเมินผล — โจทย์ฝึกพัฒนา

โจทย์ทั้ง 5 ข้อต่อยอดจากโค้ดใน Step 1–4 ทุกข้อแก้ในไฟล์เดียวคือ lib/main.dart ให้นักศึกษาลองหาจุดแก้ไขเองโดยใช้ ไกด์หาจุดแก้ไข ด้านล่าง แล้วเทียบผลกับ ม็อคอัพผลลัพธ์ที่ต้องการ ในตอนท้าย

  1. สวิตช์ 3 โหมด — เพิ่มสวิตช์ใน AppBar ให้ผู้ใช้สลับได้ 3 ค่า คือ Light, Dark, System โดยใช้ PopupMenuButton หรือ SegmentedButton
  2. textTheme — เพิ่ม textTheme ลงใน ThemeData กำหนด titleLarge, bodyMedium, labelSmall ให้มีขนาด/น้ำหนักของตัวเอง แล้วนำไปใช้ในการ์ดแทน TextStyle แบบดิบ
  3. เปลี่ยน seedColor ตอนรันไทม์ — เพิ่มแถวจุดสีให้เลือก (เช่น Teal, Indigo, Orange, Pink) เมื่อกดให้สร้าง ColorScheme.fromSeed ด้วยสีใหม่ แล้วทั้งแอปเปลี่ยนสีทันที (เหมือนการกดแถบสีในม็อคอัพด้านบน)
  4. ใช้ role สีเพิ่ม — เพิ่มวิดเจ็ตใหม่ 1 ตัว เช่น Chip หรือแถบสถานะ ที่ใช้คู่สี secondaryContainer + onSecondaryContainer เพื่อฝึกใช้ role สีอื่นนอกจาก primary
  5. ผูกปุ่ม FAB ให้ทำงาน (ท้าทาย) — ทำให้ปุ่ม FloatingActionButton ที่ตอนนี้ว่างอยู่ สุ่ม seedColor ใหม่ทุกครั้งที่กด แล้วรีบิลด์ธีม (ใช้ร่วมกับข้อ 3)
